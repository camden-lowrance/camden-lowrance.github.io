01

Current Work

Beck's Hybrids

At Beck's, I built the modern farm data pipeline from scratch, replacing older server-heavy GIS workflows with cloud-native Python services.

It is core infrastructure for the farm record, including harvest, planting, application, tillage, and other machine-generated field operations. The system has processed more than 500 billion rows of spatial telemetry. If the data is late or wrong, the rest of the product has nothing solid to build on.

I currently serve as the tech lead for a small team, so the work is not just implementation. I set direction, unblock delivery, and stay close to the production path from ingestion through storage, cleanup, and issue response.

My day-to-day work spans synchronous APIs, asynchronous workers, storage formats, reliability, and incident response. That foundation is also what makes newer workflows useful for me: they increase throughput because the system and the domain are already clear in my head.

How I Work Next

Speech-First, Tighter Loops

I currently prefer a speech-first approach with tools like Codex and Claude Code because it lets me work at the level of intent, constraints, and outcomes instead of just keystrokes. I think a lot of software is collapsing toward direct interaction: fewer screens, fewer clicks, and a shorter distance between a user's need and the data or action that answers it.

Early Adoption

New Tools, Used Early

I signed up for GitHub Copilot beta as soon as it was available, and I keep testing newer tools and workflow changes as they appear. The specifics will keep changing, so I stay focused on the underlying shift.

Domain Leverage

Throughput With Context

AI tools increase my throughput because they are paired with domain knowledge across farm data, spatial systems, APIs, and async processing. Generic tooling is not enough on its own. The value comes from knowing what to ask for, what to reject, and how the system actually works.

Continuous Involvement

From Start to Finish

I am moving toward a more continuous role in software delivery, staying involved from the first user need through implementation, refinement, and production feedback. I still write code, but more of my value now comes from keeping that loop tight and making the system more responsive to what users actually need.
