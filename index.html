Current Role

Beck's Hybrids and the core FARMserver data path.

Controller-Data Rewrite

The previous path relied on Windows servers and proprietary ArcGIS workflows. I rebuilt that core processing path as an open-source Python system designed for Kubernetes-native, cloud-native operation.

This is the pipeline that transforms harvest, planting, application, tillage, and other machine-generated field operations into the core farm record used across the product.

Scale and Data Design

The pipeline handles high-volume, high-dimensional precision agriculture data in production. My work has centered on making it scalable, reliable, and maintainable under real operating pressure.

The stack uses open geospatial and analytical formats suited for cloud processing, including GeoParquet, Parquet, cloud optimized GeoTIFFs, shapefiles, GeoJSON, and object storage patterns.
